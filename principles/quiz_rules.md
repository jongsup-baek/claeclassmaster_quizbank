# 퀴즈 슬라이드 작성 원칙

> 강의 퀴즈 슬라이드 작성 포맷 및 원칙.
>
> 출처: class_svbasic class02 퀴즈 작성 경험 기반

---

## 파일 네이밍

- 퀴즈 슬라이드: `class{NN}_Q_{영문주제}_marp.md`

## 페이지 구조

- 문제 1개 = 2페이지 (문제 페이지 + 해답 페이지)
- 타이틀 형식: `## Quiz 🔖 X.Y 절제목 (N/M) (문제|해답)`
  - X.Y: 강의번호.섹션번호
  - (N/M): 해당 절의 N번째 문제 / 총 M개
  - (문제): 출제 페이지, (해답): 정답+해설 페이지

## 문제 페이지 템플릿

```markdown
## Quiz 🔖 X.Y 절제목 (N/M) (문제)

📌 문제 N. 질문 텍스트 (한 문장)

&emsp;1\) 선택지 1
&emsp;2\) 선택지 2
&emsp;3\) 선택지 3
&emsp;4\) 선택지 4

✍️ 문제를 풀어보세요
```

## 해답 페이지 템플릿

```markdown
## Quiz 🔖 X.Y 절제목 (N/M) (해답)

📌 문제 N. 질문 텍스트 (한 문장, 문제 페이지와 동일)

&emsp;1\) 선택지 1
&emsp;2\) 선택지 2
&emsp;3\) 선택지 3
&emsp;4\) 선택지 4

✅ 정답: N
⭐ 난이도: 하/중/상

💡 해설:
- 첫 번째 포인트 (핵심 개념 설명)
- 두 번째 포인트 (구체적 예시나 비교)
- 세 번째 포인트 (실무 관점 또는 주의사항)
```

## 이모지 규칙

| 용도 | 이모지 |
|------|:------:|
| 문제 | 📌 |
| 풀어보세요 | ✍️ |
| 정답 | ✅ |
| 난이도 | ⭐ |
| 해설 | 💡 |
| 절 번호 (타이틀) | 🔖 |

## 마크다운 문법

- 선택지 번호: `1\)` (괄호 이스케이프, Marp ordered list 방지)
- 선택지 들여쓰기: `&emsp;` (HTML em space)
- 문제 텍스트: `📌 문제 N.` + 질문을 한 줄로 병합 (두 줄 금지)
- 블릿(`-`) 사용하지 않음 — 문제/정답/난이도/풀어보세요는 이모지로 대체
- 해설 본문은 블릿(`-`) 리스트로 문장 구분
- 코드 블록 포함 문제: 문제 텍스트 뒤 코드 블록, 선택지는 동일 형식

## 빈 줄 규칙

- 문제 ↔ 선택지: 빈 줄 1개
- 선택지 ↔ 풀어보세요: 빈 줄 1개
- 문제 페이지 ↔ 해답 페이지: 페이지 구분 `---`
- 정답+난이도 ↔ 해설: 빈 줄 1개
- 💡 해설: ↔ 블릿 본문: 줄바꿈 없이 바로 다음 줄

## 해설 작성 원칙

- 3-4문장, 블릿(`-`)으로 각 문장 구분
- 단순 정답 반복 금지, 배경 지식과 이유를 포함
- 관련 타입/키워드를 괄호로 나열하여 학습 연결
- 실무 관점의 의미나 주의사항 포함 권장

## 난이도 기준

| 난이도 | 판별 조건 | 예시 |
|:------:|-----------|------|
| 하 | 단일 개념의 정의/용어 질문. 코드 없음. 강의 노트에 답이 그대로 있음 | "2-state 타입이 가질 수 있는 값은?" |
| 중 | 2개 이상 개념 비교, 또는 짧은 코드 읽기(5줄 이하). 선택지 간 차이를 이해해야 답 가능 | "Variable과 Net의 역할 차이는?" |
| 상 | 코드 실행 결과 예측, 버그 찾기, 또는 3개 이상 개념 조합. 코드 6줄 이상 | "다음 코드의 출력 결과는?" |

난이도 판별 체크리스트:

- 코드가 없으면 → 하 또는 중
- 코드가 있고 실행 결과를 물으면 → 상
- "다음 중 올바른 것"류 단순 선택 → 하
- "차이점", "비교" 키워드 → 중
- 복수 개념 조합 필요 → 상

## 특수 페이지

- 마지막 페이지: `## Thank You` (내용 없음)
- 본 강의 퀴즈 마지막: Thank You + 절 목록 나열 불필요
